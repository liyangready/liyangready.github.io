ECMASCript规范关于this的说明

发表于 2015-11-21   |   分类于 前端   |  

javascript中的this一直是很多初学者搞不清的地方，最后总结出各种‘经验’，我们还是从原理出发，看看官方规范是怎么解释this这件事情的。

全局代码

规范 , this 指向 Global Environment 就是 window，这点大家都知道了，没什么好解释的。

函数代码

规范如是说：

The following steps are performed when control enters the execution context for function code contained in function object F, a caller provided thisArg, and a caller provided argumentsList:

从上面可以获取到，在执行函数代码时，this作为参数，是由调用者提供的，我们看调用的时候thisArg是如何传递进去的。

function call

The production CallExpression : MemberExpression Arguments is evaluated as follows:

  1. Let ref be the result of evaluating MemberExpression.
  2. Let func be GetValue(ref).
  3. Let argList be the result of evaluating Arguments, producing an internal list of argument values (see11.2.4).
  4. If Type(func) is not Object, throw a TypeError exception.
  5. If IsCallable(func) is false, throw a TypeError exception.
  6. If Type(ref) is Reference, then
    1. If IsPropertyReference(ref) is true, then**
      1. Let thisValue be GetBase(ref).
    2. Else, the base of ref is an Environment Record
      1. Let thisValue be the result of calling the ImplicitThisValue concrete method of GetBase(ref).
  7. Else, Type(ref) is not Reference.
    1. Let thisValue be undefined.
  8. Return the result of calling the [[Call]] internal method on func, providing thisValue as the this value and providing the list argList as the argument values.

看上面第6点，写的很清楚，在函数调用的时候，如果调用的是reference类型，那么this指向GetBase(ref). 要搞清楚这个问题，就要先看看reference到底是个啥。

reference type

reference是一个规范类型，也就是说是为了解释规范某些行为而存在的，比如delete、typeof、赋值语句等。一个reference是用来解释标示符绑定的，它由3部分组成，base、referenced name 和 用来标示 是否为严格模式的 strict reference flag.

base就是记录reference的依赖项，referenced name很好理解就是引用名称。既然是个规范类型，也没有太多的经验可以遵循，关于 base、referenced name的值我们都应该根据规范定义来判断。

我们举几个例子：

  • 比如根据规范定义，标示符引用会返回一个reference ，它的base是是一个环境记录项：

所以如果调用 foo(); 根据上面的规范说明：

Else, the base of ref is an Environment Record

  1. Let thisValue be the result of calling the ImplicitThisValue concrete method of GetBase(ref).

会走到这个else里面，于是，this的值等于环境记录的隐式this值，又有规范说了

ImplicitThisValue is always set to undefined

隐式this永远指向 undefined，所以 foo(); 执行的时候 this = undefined;

  • 再比如一个对象的属性访问也会返回一个 reference：

那么 foo.bar(); 执行时根据上面的规范，会走到这个 if 中：

If IsPropertyReference(ref) is true, then**

  1. Let thisValue be GetBase(ref).

于是真相大白，此时 this = GetBase(ref) = foo;

规律

由上面的情况我们得知，如何判断一个 this 指向，关键在于函数调用的时候，括号左边的表达式是否是一个 reference 对象，我们只需要查找规范判断这个，任何this问题都能搞定，比如很奇葩的这种 (f = foo.bar)() ， this情况如何计算？

我们需要去查找 赋值语句 的规范：

  1. Let lref be the result of evaluating LeftHandSideExpression.
  2. Let rref be the result of evaluating AssignmentExpression.
  3. Let rval be GetValue(rref).
  4. Throw a SyntaxError exception if the following conditions are all true:
    • Type(lref) is Reference is true
    • IsStrictReference(lref) is true
    • Type(GetBase(lref)) is Environment Record
    • GetReferencedName(lref) is either “eval” or “arguments”
  5. Call PutValue(lref, rval).
  6. Return rval.

看第3 行，结果返回的是 rval，rval又是经过 GetValue 函数求值，从而 foo.bar 本身是一个reference，求值之后变成了一个普通的 object，再套到之前的规范公式，不是reference，this = undefined.

同理，假设是 (1 , foo.bar)()，只需要看 逗号表达式会返回什么？

  1. Let lref be the result of evaluating Expression.
  2. Call GetValue(lref).
  3. Let rref be the result of evaluating AssignmentExpression.
  4. Return GetValue(rref).

发现和赋值语句一样，所以结果 this = undefined;

统计

有大牛统计的表格，关于常见 javascript语句是否会返回reference：

Example Reference? Notes
“foo” No
123 No
/x/ No
({}) No
(function(){}) No
foo Yes Could be unresolved reference if foo is not defined
foo.bar Yes Property reference
(123).toString Yes Property reference
(function(){}).toString Yes Property reference
(1,foo.bar) No Already evaluated, BUT see grouping operator exception
(f = foo.bar) No Already evaluated, BUT see grouping operator exception
(foo) Yes Grouping operator does not evaluate reference
(foo.bar) Yes Ditto with property reference

注意：

上面说的 thisValue = undefined； 是指的最后函数可执行上下文执行函数调用时候的 thisArg ， 规范有一句话是：

  1. Else if thisArg is null or undefined, set the ThisBinding to the global object.

如果thisArg 是 undefined，最后代码运行中的this绑定是 全局对象。
